# PR: 保存処理のUX改善 (feat/improve-save-ux)

## 概要
日程保存時のユーザー体験を改善しました。
従来の「トースト通知＋自動遷移」から、「全画面ローディングオーバーレイ」による明確なフィードバックに変更し、ユーザーが処理状況を直感的に理解できるようにしました。

## 変更点

### Frontend
- **共通コンポーネント**:
  - `src/app/components/ui/loading-overlay.tsx`: 画面全体を覆うローディングコンポーネントを新規作成。半透明の背景とスピナー、メッセージを表示。
  - ディレクトリ構造を見直し、再利用可能なUIパーツとして `src/app/components/ui/` に配置しました。

- **結果画面 (`src/app/pages/result.tsx`)**:
  - イベント確定処理（計算中）の表示を、既存のインライン表示から `LoadingOverlay` に変更・統一しました。
  - メッセージ: "最適なスケジュールとお店を計算しています..."

- **トップページ (`src/app/components/HeroSection.tsx`)**:
  - イベント作成処理中に `LoadingOverlay` を表示するように変更。

- **管理者画面 (`src/app/pages/admin.tsx`)**:
  - 保存ボタン押下時に `LoadingOverlay` を表示するように変更。
  - 処理完了のトースト通知を削除（誤解を招く可能性があるため）。
  - ※ 完了後のシェアモーダル表示と、OKボタンによる画面遷移は維持。

- **参加者画面 (`src/app/pages/Participant.tsx`)**:
  - 保存ボタンのテキストを「回答を保存して次へ」から「回答を保存」に変更。
  - 保存ボタン押下時に `LoadingOverlay` を表示するように変更。
  - 処理完了のトースト通知（ポップアップ）を削除。
  - ※ 保存完了直後の結果画面への自動遷移は維持。

## 確認方法

1.  **イベント作成**:
    - トップページからイベントを作成し、ローディングオーバーレイが表示されることを確認。
2.  **管理者として**:
    - イベント作成 -> 日程入力 -> 「日程を確定して保存」をクリック。
    - 画面中央に「日程を保存しています...」オーバーレイが表示されることを確認。
    - 完了後、シェアURLモーダルが表示されることを確認。
3.  **参加者として**:
    - URLからアクセス -> 名前入力/日程選択 -> 「回答を保存」をクリック。
    - 画面中央に「回答を保存しています...」オーバーレイが表示されることを確認。
    - 完了後、自動的に結果画面へ遷移することを確認。
